TaskFlow
Problem Solved

I was using sticky notes and random files to track tasks — total chaos. Existing apps were either too complex or didn't save state. I needed something minimal, fast, and persistent.

Approach

Designed the data model first: each task has an ID, title, priority, category, and completion state. Used localStorage with JSON serialization to persist between sessions. Built a MVC-style separation — data layer, render layer, event layer.

Key Logic
taskflow.js — core save/load 
// Serialize tasks to localStorage
const saveTasks = (tasks) => {
  localStorage.setItem('taskflow_data', JSON.stringify(tasks));
};

const loadTasks = () => {
  try {
    return JSON.parse(localStorage.getItem('taskflow_data')) ?? [];
  } catch { return []; }
};

// Render only changed nodes — no full re-render
const diffRender = (prev, next) => {
  next.forEach((task, i) => {

  });
};

Calculator
Problem Solved

Every tutorial calculator breaks on chained operations: 3 + 4 × 2. Mine handles operator precedence correctly, supports keyboard input, and deals with edge cases like dividing by zero gracefully.

Key Logic
calculator.js — eval engine 
// Safe expression evaluator (no eval())
function calculate(expr) {
  const tokens = tokenize(expr);
  const rpn    = shuntingYard(tokens); // Dijkstra
  return evalRPN(rpn);
}

// Handle division-by-zero explicitly
if (op === '/' && b === 0) return 'ERR: ÷0';

// Keyboard support
document.addEventListener('keydown', e => {
  if (e.key === 'Enter') calculate(display.value);
  if ('0123456789+-*/.'.includes(e.key)) append(e.key);
});
Features

Shunting-yard algorithm for operator precedence · Full keyboard support · Division-by-zero error handling · History log · Responsive grid layout

Approach

Instead of memorizing solutions, I rebuild data structures from scratch in C++ — understanding every pointer, every edge case. This forces genuine understanding over superficial recall.

Key Implementation
linked_list.cpp — custom implementation 
template<typename T>
class LinkedList {
  struct Node {
    T data; Node* next = nullptr;
    Node(T val) : data(val) {}
  };
  Node* head = nullptr;

public:
  void push(T val) {
    Node* n = new Node(val);
    if (!head) { head = n; return; }
    Node* cur = head;
    while(cur->next) cur = cur->next;
    cur->next = n;
  }
  // O(n) traversal — no shortcuts
  void print() {
    for(auto c = head; c; c = c->next)
      std::cout << c->data << " → ";
  }
};
